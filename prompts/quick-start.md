# 5 分钟上手提示词模板

## 1. 代码生成

```markdown
你是资深 [语言] 开发者。

任务：[描述要生成的代码]

要求：
- 完整可运行
- 添加注释
- 包含使用说明
- 列出依赖

输出完整代码。
```

---

## 2. 代码审查

```markdown
审查以下代码的安全性和质量问题：

```[语言]
[粘贴代码]
```

重点检查：
1. 安全漏洞
2. 性能问题
3. 代码规范

输出格式：
- 🟢 优点
- 🔴 严重问题
- 🟡 建议改进
```

---

## 3. Bug 修复

```markdown
修复这个 bug：

【现象】
[描述 bug 现象]

【错误信息】
[粘贴错误日志]

【相关代码】
```[语言]
[粘贴代码]
```

找出根本原因并修复。
```

---

## 4. 写技术教程

```markdown
写一篇关于 [主题] 的技术教程。

要求：
- 读者：[初学者/中级/高级]
- 字数：[2000-3000]
- 包含代码示例
- 结构清晰

大纲：
1. 简介
2. 前置条件
3. 步骤详解
4. 常见问题
5. 总结
```

---

## 5. 数据分析

```markdown
分析以下数据：

【数据】
[粘贴数据]

【问题】
[要回答的问题]

输出：
- 关键发现
- 数据可视化建议
- 行动建议
```

---

## 6. 邮件/文档写作

```markdown
帮我写一封 [类型] 邮件/文档。

【背景】
[情况说明]

【目的】
[要达成什么]

【要点】
- [要点 1]
- [要点 2]

语气：[正式/友好/简洁]
```

---

## 7. 学习计划

```markdown
我要在 [时间] 内学会 [技能]。

【当前水平】
[描述当前水平]

【目标】
[学完后能做什么]

【可用时间】
每天 [X] 小时

请制定详细学习计划，包括：
- 每天学习内容
- 实战项目
- 资源推荐
- 检验方法
```

---

## 8. 问题诊断

```markdown
我遇到了一个问题：

【现象】
[描述现象]

【已尝试】
- [尝试 1]
- [尝试 2]

【环境】
[系统/版本等]

请帮我诊断并给出解决方案。
```
