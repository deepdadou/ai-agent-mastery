## 🧠 核心思维

### 1. AI 不是搜索引擎

❌ 错误用法：
```
"什么是区块链？"
"怎么学 Python？"
```

✅ 正确用法：
```
"我要在 30 天内学会 Python 并找到兼职，请给我制定详细计划，包括：
- 每天学习内容和时长
- 实战项目建议
- 接单平台推荐
- 预期收入"
```

### 2. 把 AI 当员工，不是当工具

**工具思维：** 我让 AI 做什么，它做什么
**员工思维：** 我给 AI 目标、资源、反馈，它给我结果

```
❌ "写一个登录页面"
✅ "你是资深前端工程师，负责我们项目的登录模块。
     需求：支持邮箱/手机登录，包含验证码，适配移动端
     技术栈：React + TailwindCSS
     请输出完整代码和测试用例"
```

### 3. 迭代优于完美

不要指望一次就完美，要会迭代：

```
第 1 轮：生成基础版本
第 2 轮：指出问题，要求改进
第 3 轮：添加细节和优化
第 4 轮：最终审查
```

**关键：** 每次迭代都要给出具体反馈

## 🎯 提示词终极法则

### MASTER 法则

```
M - Model（模型选择）：选对 AI 模型
A - Aim（目标明确）：说清楚要什么
S - Structure（结构化）：用结构化的提示词
T - Task（任务分解）：大任务拆小
E - Example（示例）：给示例效果更好
R - Refine（迭代优化）：不断反馈改进
```

### 万能提示词模板

```markdown
# 角色设定
你是 [具体角色]，擅长 [具体技能]，有 [X 年] 经验

# 背景
[任务背景，越具体越好]

# 目标
[要明确达成的目标]

# 任务
[具体要做什么，分点列出]

# 要求
- [要求 1]
- [要求 2]
- [要求 3]

# 约束
- [约束 1，如字数、格式、时间等]
- [约束 2]

# 输出格式
[希望 AI 以什么格式输出]

# 示例（可选）
[给一个参考示例]
```

### 实战示例

```markdown
# 角色设定

# 示例
| 平台 | 特点 | 适合类型 | 预期收入 |

## 🛠️ 平台操作指南

### OpenClaw

**特点：** 本地部署，隐私好，可定制
**适合：** 自动化任务、长期运行

**核心技能：**
```bash
# 查看可用技能
openclaw skills list

# 安装技能
openclaw skills install <skill-name>

# 运行技能
openclaw skills run <skill-name> --args "参数"
```

**最佳实践：**
1. 用 HEARTBEAT.md 设置定时任务
2. 用 memory/ 目录做持久化
3. 用 TOOLS.md 记录配置信息

### Claude

**特点：** 代码能力强，逻辑清晰
**适合：** 代码生成、复杂任务

**技巧：**
```
1. 用项目（Projects）功能保存上下文
2. 长任务用 Artifacts 输出代码
3. 复杂任务分多个对话处理
```

### 扣子 (Coze)

**特点：** 中文友好，插件多
**适合：** 内容创作、自动化工作流

**技巧：**
```
1. 用工作流串联多个 Bot
2. 利用插件扩展能力
3. 设置定时触发
```

### 文心一言

**特点：** 中文理解好，免费额度多
**适合：** 中文内容、日常问答

**技巧：**
```
1. 用 ERNIE Bot 4.0 处理复杂任务
2. 利用插件市场扩展功能
3. 注意免费额度限制
```

### Kimi

**特点：** 长文本处理好
**适合：** 文档分析、长文写作

**技巧：**
```
1. 直接上传 PDF/Word 文档
2. 支持 200 万字上下文
3. 适合做文献综述
```

## 🎯 实战场景

### 场景 1：GitHub Bounty 自动化

**目标：** 自动找任务、生成解决方案

**工作流：**
```
1. 用脚本抓取 GitHub bounty issue
2. 用 Claude 分析并生成解决方案
3. 人工审查后提交 PR
4. 等待 bounty 到账
```

**提示词：**
```markdown
分析这个 GitHub issue，生成解决方案：

【Issue】
[粘贴内容]

【要求】
1. 用一句话总结问题核心
2. 生成最小可行解决方案
3. 代码要简洁便于 review
4. 添加测试用例
5. 写 PR 描述

【输出】
- 问题分析
- 修改的文件
- 完整代码
- 测试代码
- PR 描述模板
```

### 场景 2：智能合约审计

**目标：** Code4rena 竞赛获奖

**工作流：**
```
1. 获取合约代码
2. 用 Claude 深度审计
3. 整理漏洞报告
4. 提交到 Code4rena
```

**提示词：**
```markdown
审计这个智能合约的安全漏洞：

【合约代码】
[粘贴代码]

【审计重点】
1. 重入攻击
2. 整数溢出
3. 权限控制
4. 价格操纵
5. 闪电贷攻击

【输出格式】
对每个漏洞：
- 严重程度 (High/Medium/Low)
- 位置和描述
- 利用场景
- 修复建议和 PoC

【最后】
- 漏洞统计表
- 安全评分 (1-10)
```

### 场景 3：内容创作赚 USDG

**目标：** Superteam 内容 bounty

**工作流：**
```
1. 找 Superteam 内容任务
2. 用 AI 生成初稿
3. 人工润色和事实核查
4. 发布并提交
```

**提示词：**
```markdown
写一篇关于 [主题] 的技术教程：

【要求】
- 目标读者：[初学者/中级/高级]
- 字数：2000-3000 字
- 包含代码示例
- 结构清晰有目录

【大纲】
1. 简介（解决什么问题）
2. 前置条件
3. 步骤详解（每步有代码）
4. 常见问题
5. 总结

【风格】
- 简洁明了
- 代码有注释
- 有截图位置提示
```

### 场景 4：批量处理任务

**目标：** 同时处理多个 bounty

**工作流：**
```
1. 收集所有任务到一个文件
2. 用脚本批量调用 AI
3. 整理输出结果
4. 逐个提交
```

**脚本示例：**
```bash
#!/bin/bash
# 批量审计合约

for contract in contracts/*.sol; do
  echo "审计 $contract..."
  claude "审计这个合约：$(cat $contract)" >> reports/$(basename $contract).md
done

echo "所有审计完成！"
```

## ⚡ 效率翻倍技巧

### 1. 创建提示词库

```bash
mkdir ~/prompts

# 代码审查
cat > ~/prompts/code-review.md << 'EOF'
审查以下代码的安全性：
```$CODE
```

重点检查：
1. SQL 注入
2. XSS 攻击
3. 认证问题
4. 数据泄露
EOF

# 使用时
sed "s|\$CODE|$(cat mycode.py)|" ~/prompts/code-review.md | claude
```

### 2. 用变量模板

```markdown
# 通用 Bug 修复模板

修复这个 bug：
- 错误信息：{{ERROR}}
- 相关代码：{{CODE}}
- 已尝试：{{ATTEMPTS}}
- 环境：{{ENV}}
```

### 3. 链式调用

```bash
# 第一步：生成代码
claude "生成一个 Python 脚本，功能是 XXX" > output.py

# 第二步：审查代码
claude "审查这个代码的安全问题：$(cat output.py)" >> review.md

# 第三步：生成测试
claude "为这个代码写测试：$(cat output.py)" > test.py
```

### 4. 并行处理

```bash
# 同时处理多个任务
claude "任务 1..." &
claude "任务 2..." &
claude "任务 3..." &
wait
echo "所有任务完成！"
```

### 5. 保存上下文

```bash
# 保存对话历史
claude "..." >> session.md

# 下次继续
claude "基于之前的讨论：$(cat session.md)，继续..."
```

## ⚠️ 避坑指南

### 1. 不要过度依赖

❌ 完全让 AI 写代码，不审查就提交
✅ AI 生成初稿，人工审查后提交

### 2. 不要一次性给太多

❌ 一次让 AI 做 10 件事
✅ 拆分成小任务，逐个完成

### 3. 不要忽略上下文限制

❌ 超过模型 token 限制还继续对话
✅ 定期总结，开始新对话

### 4. 不要相信 AI 说的所有话

❌ AI 说的代码直接运行
✅ 一定要测试验证

### 5. 不要忽略成本

❌ 无节制使用付费 API
✅ 根据任务复杂度选择模型

## 🚀 进阶技巧

### 1. 思维链（Chain of Thought）

让 AI 先思考再回答：
```markdown
请先逐步分析问题，然后：
1. 列出可能的解决方案
2. 分析每个方案的优缺点
3. 选择最优方案并说明理由
4. 生成具体实现
```

### 2. 角色扮演

```markdown
你是有 10 年经验的 [角色]，
曾 [成就 1]、[成就 2]，
擅长 [技能]。

请 [任务描述]...
```

### 3. 对比分析

```markdown
对比以下两种方案：
方案 A: [描述]
方案 B: [描述]

从以下维度对比：
- 性能
- 成本
- 可维护性
- 风险

推荐最优方案并说明理由
```

### 4. 增量开发

```markdown
我们分 5 轮完成这个项目：

第 1 轮：需求分析和架构设计
第 2 轮：核心功能实现
第 3 轮：错误处理和日志
第 4 轮：测试用例
第 5 轮：文档和优化

现在开始第 1 轮...
```
